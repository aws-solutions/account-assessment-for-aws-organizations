Migration Risk Scanner — source org accounts
This is where you find what breaks Scan accounts in the source (old) org to discover resource-based policies that reference organization-specific conditions. These policies will silently stop working after migration because the account's org ID changes.
  • aws:PrincipalOrgID / aws:PrincipalOrgPaths — condition checks the caller's org membership. Fails when the account moves to a different org.
  • aws:ResourceOrgID / aws:ResourceOrgPaths — condition checks the resource's org. Same issue.
  • Org ID literals (e.g. o-xxxxxxxxxx) hardcoded anywhere in the policy.
Remediation: Before migration, update flagged policies to include both old and new org IDs, or replace org conditions with explicit account-based principals. Prioritize KMS key policies and S3 bucket policies — broken access can lock you out of encrypted data.
